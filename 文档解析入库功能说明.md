# 文档解析和入库功能说明

## 功能概述

新增了完整的文档解析层，支持用户上传 PDF、Word、Excel、图片等文件，自动解析为纯文本后进行切片、元数据生成，最终入库到 Milvus。

## 完整流程

```
用户上传文件 
  → DocumentParserService（文档解析）
    → 根据文件类型路由到对应解析器
      → PdfParser / WordParser / ExcelParser / ImageOcrParser
        → 提取纯文本内容
          → DocumentIngestionService（文档入库服务）
            → ChunkMetadataService（切片+元数据生成）
              → MilvusChunkRepository（入库到Milvus）
```

## API接口说明

### 1. 单文件上传

**接口**: `POST /api/rag/upload`

**参数**:
- `file` (必填): 上传的文件
- `chunkSize` (可选): 切片大小，默认500字符

**请求示例** (curl):
```bash
curl -X POST http://localhost:8080/api/rag/upload \
  -F "file=@外汇管理政策.pdf" \
  -F "chunkSize=500"
```

**响应示例**:
```json
{
  "success": true,
  "docId": "doc_外汇管理政策_1718600000000_a1b2c3d4",
  "fileName": "外汇管理政策.pdf",
  "chunkCount": 15,
  "textLength": 7500,
  "processingTime": 1234,
  "message": "文档入库成功"
}
```

### 2. 批量文件上传

**接口**: `POST /api/rag/upload/batch`

**参数**:
- `files` (必填): 上传的多个文件
- `chunkSize` (可选): 切片大小，默认500字符

**请求示例** (curl):
```bash
curl -X POST http://localhost:8080/api/rag/upload/batch \
  -F "files=@文件1.pdf" \
  -F "files=@文件2.docx" \
  -F "files=@文件3.xlsx"
```

**响应示例**:
```json
{
  "success": true,
  "totalCount": 3,
  "successCount": 2,
  "failCount": 1,
  "results": [
    {
      "fileName": "文件1.pdf",
      "success": true,
      "docId": "doc_文件1_xxx",
      "chunkCount": 10
    },
    {
      "fileName": "文件2.docx",
      "success": true,
      "docId": "doc_文件2_xxx",
      "chunkCount": 8
    },
    {
      "fileName": "文件3.xlsx",
      "success": false,
      "message": "解析失败原因"
    }
  ]
}
```

### OCR配置（图片识别）

Tesseract OCR 需要安装 Tesseract 引擎并配置语言数据：

1. **Windows**: 下载并安装 [Tesseract](https://github.com/UB-Mannheim/tesseract/wiki)
2. **设置环境变量**: `TESSDATA_PREFIX` 指向 tessdata 目录
3. **下载语言数据**: 确保包含 `chi_sim.traineddata` (简体中文) 和 `eng.traineddata` (英文)

如果未配置 Tesseract，图片OCR功能将不可用，但不影响其他文件格式的解析。

## 使用示例

### Java代码调用

```java
@Autowired
private DocumentIngestionService documentIngestionService;

public void uploadAndIngest(MultipartFile file) {
    // 使用默认切片大小500
    IngestionResult result = documentIngestionService.ingestFile(file);
    
    // 或指定切片大小
    IngestionResult result = documentIngestionService.ingestFile(file, 300);
    
    System.out.println("文档ID: " + result.getDocId());
    System.out.println("切片数量: " + result.getChunkCount());
}
```

### 前端调用示例

```javascript
const formData = new FormData();
formData.append('file', fileInput.files[0]);

fetch('/api/rag/upload?chunkSize=500', {
  method: 'POST',
  body: formData
})
.then(response => response.json())
.then(data => {
  if (data.success) {
    console.log('入库成功，文档ID:', data.docId);
  }
});
```

## 注意事项

1. **文件大小限制**: 默认单个文件最大50MB，可在 application.yml 中调整
2. **文档ID生成规则**: `doc_文件名_时间戳_UUID前8位`
3. **切片大小建议**: 
   - 短文档（<5000字符）: 300-500字符
   - 长文档（>5000字符）: 500-800字符
   - 表格类文档: 200-300字符
4. **OCR识别**: 图片质量会影响识别准确率，建议使用清晰的扫描件
5. **Excel解析**: 表格数据会转换为 `列1 | 列2 | 列3` 的文本格式

## 解决的问题

✅ 实现了文档解析层，支持多种文件格式
✅ 统一入库流程：解析 → 切片 → 元数据生成 → Milvus入库
✅ 保留文档结构信息（标题层级、段落边界）
✅ 自动生成完整的元数据（seg_id/doc_id/content/pos/pre_seg_id/next_seg_id/tags）
✅ 语义割裂检测，提高检索准确率
✅ 同一份文件上传到不同知识库效果一致（基于统一的解析和切片标准）
